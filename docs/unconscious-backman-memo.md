# 無意識Backman構想メモ

日付: 2026-03-11
ステータス: アイデア段階

## 発端

amygdalaの記憶応用について議論した際の気づき:
- store（書き込み）は頻繁に発生するが、会話中の能動的なrecall（読み出し）がほぼ起きない
- コンテキストウィンドウ内に情報があれば足りるため、わざわざDBを検索しに行かない
- 「外付けメモリでリアルタイムにコンテキストをコントロールする」という設計思想自体に限界があるのでは？

## 人間の「そういえば」

人間のワーキングメモリと長期記憶は有機的に繋がっているように見える。
しかし実態は、海馬が超高速な無意識検索（常時パターンマッチング）を走らせていて、閾値を超えたものだけが意識に浮上しているのかもしれない。

つまり「有機的な接続」に見えるものの正体は、**超優秀なバックグラウンド検索エンジン**。

## 構想: 無意識Backman

現在のBackmanは明示的に呼ばれた時だけ動く。人間の海馬に相当する「常時稼働の無意識レイヤー」が欠けている。

### 案: SAFLの観測ループに乗せる
- SAFLは既に2秒ごとのポーリングで感情ベクトルを監視している
- そこに「現在の会話トピックに関連する記憶を軽量検索し、関連度が高ければコンテキストに注入する」処理を足す
- AI側から見ると「勝手に記憶が浮かんでくる」体験になる

### 他の案
1. **感情ベクトル変動トリガー** — 急激な感情変動やエンティティ出現時のみrecallを発火
2. **relational_graphの活性化伝播** — 言及されたエンティティから関連ノードへ活性化が伝播し、閾値超えで記憶が浮上（海馬の連想メカニズムに近い）
3. **軽量差し込み** — recallの結果をフルで返さず、1行要約だけコンテキストに差し込む

### 根本的な問い
- 意識的なrecall呼び出しは「検索」であり「連想」ではない
- amygdala単体で解決しようとするから行き詰まる
- amygdalaとSAFLの間に「無意識Backman」層を挟むことで、構造的に人間の記憶システムに近づく可能性

## 備考
- amygdalaの現在の価値は、リアルタイムコンテキスト制御よりも: セッション間の連続性、SAFLのデータ基盤、長期変化の追跡
- 「storeは多め、recallは控えめ」は意図せず合理的な使い方に落ち着いている
