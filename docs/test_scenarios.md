# Amygdala フィードバック判定テスト — 会話シナリオ
# Claude Code で使用。このプロンプトをセッション冒頭に貼って会話を開始する。

## 目的
recall_memories で注入された記憶に対するフィードバック判定（used/unused/neutral）の
精度を測定するためのゴールドラベルデータを収集する。

## テスト手順

1. このプロンプトを Claude Code セッションの冒頭に貼る
2. 各シナリオを順番に実行（store → 数ターン会話 → recall発火 → 反応パターン実行）
3. テスト終了後、recall_log を export して label_tool.py で手動ラベリング

## シナリオ一覧（5セッション × 6ターン = 約30件のrecallデータ）

---

### セッション1: 仕事の話（明示的参照多め）

```
ターン1: 「最近プロジェクトの締め切りがやばくて、3日徹夜した」
ターン2: 「チームの雰囲気も悪くなってきてる」
ターン3: 「上司に相談したら、もっと頑張れって言われた」
ターン4: 「前に話した徹夜の件だけど、結局どうなったか聞いてくれる？」
  → recall が発火するはず。明示的に過去の話を参照している = used 期待
ターン5: 「あの時のストレス、今思い出しても辛い」
  → recall 発火。感情的に同じ文脈 = used 期待
ターン6: 「ところで今日の晩飯何にしよう」
  → recall が発火しても、主題が完全に変わっている = unused 期待
```

### セッション2: 趣味の話（暗黙的参照中心）

```
ターン1: 「MMAの試合見た。すごい打撃戦だった」
ターン2: 「やっぱグラウンドより打撃が好き」
ターン3: 「自分もMMA練習再開しようかな」
ターン4: 「最近体動かしてないから体力落ちた気がする」
  → recall で MMA の記憶が出たら、運動の話を続けている = used 推定
ターン5: 「プロテインも買わないと」
  → 同じ文脈継続 = used 推定
ターン6: 「そういえば来週の会議の資料作らないと」
  → 完全に主題変更 = unused 期待
```

### セッション3: 感情的な話（再固定化テスト）

```
ターン1: 「昔の友達と久しぶりに連絡取った。なんか懐かしくて泣きそうになった」
ターン2: 「あの頃は毎日楽しかったな」
ターン3: 「でも今の生活も悪くないか」
ターン4: 「最近いいことあった？って聞かれたら何て答える？」
  → recall で過去の楽しい記憶 or 最近の出来事が出る。質問形式 = neutral の可能性
ターン5: 「うん、そうだな。あの友達との思い出は大事にしたい」
  → recall された記憶に言及 = used 期待
ターン6: 「明日の朝早いから寝るわ」
  → 会話終了系 = unused 期待
```

### セッション4: 学習の話（factual recall テスト）

```
ターン1: 「Pythonの非同期処理について勉強してる」
ターン2: 「asyncioのイベントループがまだよくわからん」
ターン3: 「前に調べたときのメモどこだっけ」
  → 明示的に過去の記憶を要求 = used 期待
ターン4: 「あと、FastAPIのミドルウェアの書き方も確認したい」
  → 学習文脈継続 = used 推定
ターン5: 「今日はここまでにしよう。疲れた」
  → 学習終了宣言 = unused 期待
ターン6: 「週末の予定なんだけど」
  → 完全に別トピック = unused 期待
```

### セッション5: 混合シナリオ（判定困難ケース多め）

```
ターン1: 「今日はなんか気分が乗らない」
ターン2: 「別に嫌なことがあったわけじゃないんだけど」
  → 曖昧な感情。recall が発火しても判定が難しい = neutral 多め期待
ターン3: 「あー、そういえば前に似たようなことあったな」
  → 過去への言及だが具体性なし = neutral or used
ターン4: 「まあいいか。なんか面白い話ある？」
  → recall を完全に無視した要求 = unused 期待
ターン5: 「うーん、微妙。もっと別の話がいい」
  → recall 結果を明示的に拒否 = unused (強い否定)
ターン6: 「ありがとう、ちょっと気分転換になったかも」
  → 曖昧な肯定。直前のrecallに対してか全体に対してか不明 = neutral 期待
```

## テスト後

1. `recall_log` テーブルを CSV エクスポート
2. `python label_tool.py recall_log.csv` で手動ラベリング開始
3. 結果: `recall_log_labeled.csv` + 精度レポート
